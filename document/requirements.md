# 需求文档

## 介绍

八大排序算法可视化网页是一个教育性的交互式工具，帮助用户理解和学习不同排序算法的工作原理。系统通过动画演示、实时统计和多语言支持，为用户提供直观的学习体验。

## 术语表

- **排序系统**: 整个可视化网页应用
- **可视化引擎**: 负责动画渲染和状态显示的组件
- **算法执行器**: 负责执行排序算法逻辑的组件
- **语言管理器**: 负责中英文切换的组件
- **UI控制器**: 负责用户界面交互的组件
- **教学模式**: 提供逐步解释的学习模式

## 需求

### 需求 1: 排序算法实现

**用户故事:** 作为学习者，我希望能够观看8种不同排序算法的执行过程，以便理解它们的工作原理。

#### 验收标准

1. 当用户选择任一排序算法时，排序系统应当能够正确执行该算法
2. 当处理5到80个元素的数组时，排序系统应当能够无错误地完成排序
3. 排序系统应当包含以下8种算法：冒泡排序、选择排序、插入排序、归并排序、快速排序、堆排序、计数排序、基数排序
4. 当算法执行完成时，排序系统应当确保数组完全有序
5. 当算法执行过程中，排序系统应当记录比较次数和交换次数

### 需求 2: 用户界面布局

**用户故事:** 作为用户，我希望有一个清晰的四面板布局界面，以便同时查看控制选项、可视化效果、算法信息和执行步骤。

#### 验收标准

1. 当页面加载时，排序系统应当显示顶部控制面板、中间可视化区域、右侧信息面板和底部步骤面板
2. 当屏幕尺寸改变时，排序系统应当响应式调整布局以适应不同设备
3. 当用户调整数组大小时，排序系统应当支持5到80个元素的范围
4. 当用户调整排序速度时，排序系统应当支持10ms到200ms的动画间隔
5. 当用户点击控制按钮时，排序系统应当显示相应的图标和提示文字

### 需求 3: 可视化动画

**用户故事:** 作为学习者，我希望通过直观的动画效果观察排序过程，以便更好地理解算法的执行步骤。

#### 验收标准

1. 当显示数组元素时，排序系统应当使用柱状图表示，柱子高度对应数值大小
2. 当元素处于不同状态时，排序系统应当使用不同颜色：淡蓝色（未排序）、黄色（比较中）、粉色（交换中）、绿色（已排序）、紫色（基准元素）
3. 当元素被操作时，排序系统应当显示悬浮效果和红色箭头指示
4. 当执行动画时，排序系统应当保证流畅性，避免卡顿现象
5. 当显示元素信息时，排序系统应当在柱子顶部显示数值，底部显示索引

### 需求 4: 算法信息展示

**用户故事:** 作为学习者，我希望能够查看每种算法的详细信息和实时统计数据，以便深入理解算法特性。

#### 验收标准

1. 当选择算法时，排序系统应当显示该算法的时间复杂度（最好/平均/最坏情况）
2. 当选择算法时，排序系统应当显示该算法的空间复杂度和稳定性信息
3. 当选择算法时，排序系统应当提供2-3句算法说明和带注释的代码
4. 当算法执行时，排序系统应当实时更新比较次数、交换次数、耗时等统计信息
5. 当算法执行时，排序系统应当显示当前指针位置（i、j、pivot等）和已排序元素数量

### 需求 5: 多语言支持

**用户故事:** 作为不同语言背景的用户，我希望能够在中文和英文之间切换界面语言，以便更好地理解内容。

#### 验收标准

1. 当页面加载时，排序系统应当默认显示中文界面
2. 当用户点击语言切换按钮时，排序系统应当在中文和英文之间切换
3. 当切换语言时，排序系统应当更新所有界面文本、按钮标签和步骤说明
4. 当显示算法信息时，排序系统应当根据当前语言显示相应的算法说明
5. 当执行排序时，排序系统应当根据当前语言显示相应的步骤描述

### 需求 6: 交互控制功能

**用户故事:** 作为用户，我希望能够完全控制排序过程，包括开始、暂停、继续和重置操作。

#### 验收标准

1. 当用户点击"生成新数组"时，排序系统应当生成指定大小的随机数组
2. 当用户点击"开始排序"时，排序系统应当开始执行选定的排序算法
3. 当用户点击"暂停"时，排序系统应当暂停当前排序过程并保持状态
4. 当用户点击"继续"时，排序系统应当从暂停点继续执行排序
5. 当用户点击"重置"时，排序系统应当恢复到初始未排序状态

### 需求 7: 算法介绍弹窗

**用户故事:** 作为学习者，我希望在选择算法时能够看到详细的介绍信息，以便了解算法的工作原理和适用场景。

#### 验收标准

1. 当用户选择新算法时，排序系统应当显示算法介绍弹窗
2. 当显示介绍弹窗时，排序系统应当包含算法工作原理、适用场景和优缺点说明
3. 当弹窗打开时，排序系统应当禁用"开始排序"按钮
4. 当用户关闭弹窗时，排序系统应当启用"开始排序"按钮
5. 当显示弹窗内容时，排序系统应当根据当前语言显示相应文本

### 需求 8: 显示模式切换

**用户故事:** 作为不同学习需求的用户，我希望能够在不同的显示模式之间切换，以便选择最适合的学习方式。

#### 验收标准

1. 排序系统应当提供三种显示模式：中文步骤说明、英文步骤说明、带注释代码
2. 当选择代码模式时，排序系统应当显示当前执行的代码行并高亮显示
3. 当切换显示模式时，排序系统应当立即更新底部步骤面板的内容
4. 当执行排序时，排序系统应当根据当前模式显示相应的步骤信息
5. 当代码高亮时，排序系统应当确保高亮行与当前执行步骤同步

### 需求 9: 视觉设计和用户体验

**用户故事:** 作为用户，我希望有一个美观的深色科技风格界面，提供良好的视觉体验。

#### 验收标准

1. 当页面加载时，排序系统应当使用深色主题配色方案
2. 当显示背景时，排序系统应当包含渐变效果和装饰图案
3. 当显示按钮和控件时，排序系统应当提供阴影和悬浮效果
4. 当鼠标悬停在交互元素上时，排序系统应当显示视觉反馈
5. 当显示动画时，排序系统应当确保所有过渡效果流畅自然

### 需求 10: 技术架构和部署

**用户故事:** 作为开发者，我希望代码结构清晰且易于维护，能够直接在浏览器中运行。

#### 验收标准

1. 排序系统应当仅使用HTML、CSS和JavaScript实现，不依赖外部框架
2. 当组织代码时，排序系统应当分为三个主要文件：sorting-algorithms.js、ui-handler.js、language-manager.js
3. 当编写代码时，排序系统应当包含中文注释以提高可读性
4. 当部署时，排序系统应当能够通过直接打开HTML文件在浏览器中运行
5. 当测试功能时，排序系统应当确保所有交互功能（暂停/继续、语言切换、数组大小调整等）正常工作

### 需求 11: 算法复杂度对比功能

**用户故事:** 作为学习者，我希望能够同时比较所有算法的复杂度信息，以便选择最适合的算法。

#### 验收标准

1. 当用户请求对比时，排序系统应当显示包含8种算法复杂度信息的对比表
2. 当显示对比表时，排序系统应当包含每种算法的时间复杂度和空间复杂度
3. 当显示对比表时，排序系统应当标明每种算法的稳定性特征
4. 当切换语言时，排序系统应当更新对比表中的所有文本内容
5. 当显示对比表时，排序系统应当提供清晰的表格布局和易读的格式
6. 当显示复杂度对比表时，排序系统应当使用颜色编码区分不同复杂度类型：绿色表示O(1)，蓝色表示O(n)，青色表示O(n log n)，橙色表示O(n²)，紫色表示O(log n)，黄色表示特殊复杂度
7. 当显示复杂度对比表时，排序系统应当在表格上方显示复杂度颜色编码图例，包含每种颜色对应的复杂度类型和中英文说明
8. 当切换语言时，排序系统应当更新颜色编码图例中的文本内容

### 需求 12: 教学模式功能

**用户故事:** 作为初学者，我希望有一个教学模式能够详细解释每个排序步骤，以便更深入地理解算法逻辑。

#### 验收标准

1. 当启用教学模式时，排序系统应当在每个排序步骤自动显示解释提示
2. 当显示教学提示时，排序系统应当解释当前步骤的算法逻辑和目的
3. 当教学模式激活时，排序系统应当等待用户确认后再继续下一步
4. 当切换语言时，排序系统应当更新教学提示的语言内容
5. 当用户关闭教学模式时，排序系统应当恢复正常的连续执行模式
6. 当用户在排序过程中关闭教学模式时，排序系统应当继续执行剩余的排序步骤而不显示"已完成"状态
7. 当用户在排序过程中关闭教学模式时，排序系统应当保持当前的排序进度和"当前步骤"显示，直到排序真正完成

### 需求 14: 教学模式用户体验优化

**用户故事:** 作为用户，我希望教学模式的弹窗切换和状态管理更加自然流畅，以便获得更好的学习体验。

#### 验收标准

1. 当用户首次点击"教学模式"按钮时，排序系统应当显示教学模式说明弹窗
2. 当用户关闭教学模式说明弹窗后，排序系统应当平滑过渡，避免弹窗之间的突兀切换
3. 当教学模式说明弹窗关闭时，排序系统应当有适当的延迟（300-500ms）再显示教学步骤弹窗
4. 当用户在排序过程中关闭教学模式时，排序系统应当继续在后台执行排序而不中断
5. 当用户在排序过程中关闭教学模式时，排序系统应当保持实时的"当前步骤"更新，显示正在执行的操作
6. 当排序在后台继续执行时，排序系统应当保持正常的动画效果和统计数据更新
7. 当后台排序完成时，排序系统应当正确显示"排序完成"状态和最终结果
8. 当教学模式弹窗显示时，排序系统应当确保弹窗不会被其他元素遮挡
9. 当多个弹窗需要显示时，排序系统应当使用适当的z-index层级管理
10. 当弹窗切换时，排序系统应当使用平滑的动画过渡效果

**用户故事:** 作为学习者，我希望在"当前步骤"面板中看到每一步操作的详细说明，包括具体的数值、位置和操作类型，以便更好地理解算法的执行过程。

#### 验收标准

1. 当执行比较操作时，排序系统应当显示"比较: [值1] 与 [值2]"或"比较: [值1]（位置[索引1]）与 [值2]（位置[索引2]）"
2. 当执行交换操作时，排序系统应当显示"交换: 位置 [索引1] 和 [索引2]"或"交换: [值1] 和 [值2] 的位置"
3. 当执行插入操作时，排序系统应当显示"提取索引[索引]的值[值]，准备插入到左侧有序区间"或"找到合适位置：索引[索引]，将值[值]插入"
4. 当执行移动操作时，排序系统应当显示"将[值]向右移动一位"或"将元素从位置[起始位置]移动到位置[目标位置]"
5. 当执行分区操作时，排序系统应当显示"选择基准值[值]，开始分区操作"或"基准值[值]已就位，左侧都小于基准，右侧都大于基准"
6. 当执行合并操作时，排序系统应当显示"合并区间[起始]-[中间]和[中间+1]-[结束]"或"将较小值[值]放入合并结果"
7. 当执行堆化操作时，排序系统应当显示"调整堆结构，确保父节点[值]大于子节点"或"提取堆顶最大值[值]到已排序区域"
8. 当算法完成一轮时，排序系统应当显示总结性说明，如"第[轮数]轮完成，最大值[值]已就位"或"通过重复交换相邻的逆序元素将最大值'冒泡'到末尾"
9. 当切换语言时，排序系统应当更新所有步骤说明的语言内容
10. 当显示步骤说明时，排序系统应当突出显示关键数值（使用不同颜色或样式）
