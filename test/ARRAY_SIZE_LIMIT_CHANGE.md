# Array Size Limit Change

## 修改内容

将数组大小的最大限制从 **100** 改为 **70**，保持最小值为 **5** 不变。

## 修改的文件

### 1. ui-handler.js
**位置**: `changeArraySize()` 方法
```javascript
// 修改前
this.arraySize = Math.max(5, Math.min(100, size));

// 修改后  
this.arraySize = Math.max(5, Math.min(70, size));
```

### 2. index.html
**位置**: 数组大小滑块控件
```html
<!-- 修改前 -->
<input type="range" id="array-size" min="5" max="100" value="30">

<!-- 修改后 -->
<input type="range" id="array-size" min="5" max="70" value="30">
```

## 修改影响

### ✅ 保持不变的功能
- 数组大小的最小值仍为 5
- 默认数组大小仍为 30
- 滑块的基本功能和样式
- 数组生成和排序逻辑
- 可视化显示和自适应宽度计算
- 所有排序算法的正常工作
- 教学模式和显示模式切换
- 语言切换功能

### 🔄 受影响的功能
- 用户现在最多只能选择 70 个元素进行排序
- 滑块的最大值显示为 70
- 如果用户之前设置了大于 70 的值，会被自动限制为 70

## 技术细节

### 数组大小验证逻辑
```javascript
this.arraySize = Math.max(5, Math.min(70, size));
```
- `Math.max(5, ...)` 确保最小值为 5
- `Math.min(70, ...)` 确保最大值为 70
- 任何超出范围的输入都会被自动调整

### 可视化适配
现有的可视化系统已经具备良好的自适应能力：
- 柱体宽度会根据数组大小自动调整
- 容器宽度基于固定的参考尺寸计算
- 最小柱体宽度为 8px，确保即使是 70 个元素也能清晰显示
